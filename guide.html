SECTION 2

회사 GitHub 계정 만들기 (안전하게)

핵심 — "회사가 소유"하게

개인 이메일·개인 휴대폰으로만 만들면 담당자 퇴사 시 계정을 잃습니다. 공용 이메일 + 조직(Organization)으로 만드세요.

1
가입
github.com → Sign up
  • 이메일: 회사 공용 이메일(예: erp@회사도메인) — 개인 이메일 금지
  • 아이디: 회사 이름 기반(소문자·하이픈) 예: seungjeong-erp
  • 비밀번호: 12자 이상(영문 대소문자+숫자+기호) → 안전한 곳에 기록
  • 이메일 인증코드 입력 → 완료
2
2단계 인증(2FA) — 필수
GitHub는 2FA 의무화(미설정 시 로그인 차단).
  • 프로필 → Settings → Password and authentication → Enable two-factor authentication
  • 인증 앱(Google/Microsoft Authenticator)을 회사 휴대폰에 설치해 연결
  • 복구 코드(recovery codes)를 꼭 출력·보관 — 휴대폰 분실 시 유일한 복구 수단
3
회사 조직(Organization) 만들기
  • 우상단 ＋ → New organization → Free → 조직 이름(예: seungjeong)
  • 이제 ERP 저장소를 회사(조직)가 소유 → 직원 추가/제거·권한 부여 가능
SECTION 3

변경사항 올리기 (Push)

파일을 수정한 뒤 인터넷(주소)에 반영하는 방법. GitHub Desktop 사용.

1
GitHub Desktop에서 올리기
  1. GitHub Desktop 실행 → 저장소 seungjeong-erp 선택
  2. 왼쪽 아래 Summary에 바꾼 내용 한 줄 입력(예: BOM 단가 수정)
  3. Commit to main(또는 master) 클릭
  4. 위쪽 Push origin 클릭 → 2~3분 뒤 주소에 반영

※ 로그인 창이 뜨면 Sign in with your browser로 로그인하면 자동 진행됩니다.

처음 올릴 때 (저장소가 없을 때)

GitHub Desktop → File → Add local repository → 폴더 선택 → Publish repository(⚠️ "Keep this private" 체크 해제) → Settings → Pages → branch master → Save → 주소 발급.

SECTION 4

팀원이 함께 작업하기 (협업)

먼저 구분

이 협업은 프로그램 파일(화면)을 함께 고치는 담당자용입니다. 그냥 ERP를 쓰는 직원은 주소만 열면 됩니다(Section 1).

[대표님만] 협업자 초대 — 꼭 먼저

A
Collaborator 추가
  1. github.com/jachungu29/seungjeong-erp → Settings → Collaborators
  2. Add people → 팀원의 GitHub 아이디 입력 → 추가
  3. 팀원은 초대 메일/알림을 Accept(수락)

※ 이걸 안 하면 팀원이 Push(올리기)를 못 합니다.

[각 팀원 PC] 5단계

1
가입 + 초대 수락
각자 github.com 가입 → 아이디를 대표님께 전달 → 초대 수락
2
GitHub Desktop 설치·로그인
desktop.github.com 설치 → 본인 계정 로그인
3
저장소 받아오기 (Clone)
File → Clone repository → jachungu29/seungjeong-erp 선택 → Clone
4
작업 전 최신 받기 (Pull) ⭐
Fetch origin → 받을 게 있으면 Pull origin → 남들 최신 내용 내려받기 (일 시작 때마다!)
5
수정 → Commit → Push
파일 수정 → Summary 입력 → Commit → Push origin

황금 규칙 (충돌 방지)

① 일 시작 전 Pull — 항상 최신부터
② 작업 끝나면 바로 Push — 오래 들고 있지 않기
③ 같은 파일 동시 수정 금지 — 누가 어떤 파일 만질지 미리 정하기
④ Commit 메시지 꼭 적기
충돌(Conflict)이 떴다면?

당황 말고 그 화면을 캡처해 담당자(또는 Claude)에게 보여주세요. 함부로 누르지 말고 도움받아 정리.

SECTION 5

다음 단계 — Supabase 데이터 공유 (2단계)

왜 필요한가

지금은 데이터가 PC마다 따로입니다. Supabase(클라우드 DB)에 연결하면 모든 PC가 같은 BOM·데이터를 실시간 공유하게 됩니다. 이것이 대표님이 원하던 "다중 PC 연계"의 완성입니다.

1
Supabase 회사 계정 (대표님)
supabase.com 가입(공용 이메일·2FA, GitHub와 동일) → New project(지역 Northeast Asia / Seoul)
2
URL·키 전달 (대표님)
Settings → API 에서 Project URL + anon key 복사 → Claude(담당)에게 전달
3
테이블 생성 + 코드 연동 (Claude)
BOM 등 테이블 생성 → 저장을 localStorage에서 Supabase로 교체 → 로그인·RLS 보안 → Push로 반영
무료 한계 (2026)

DB 500MB · 자동백업 없음 · 1주 미사용 시 일시정지. 회사 운영은 Pro 약 $25/월 권장. 처음엔 무료로 시작 → 안정되면 전환.
